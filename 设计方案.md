# evolver.py 设计方案


## 1. 项目背景与目标

### 1.1 原始项目概述

原始仓库（Node.js 版本）位于 `./evolver/`，是 EvoMap 的 GEP (Genome Evolution Protocol) 自进化引擎。当前项目 `evolver.py` 是其 Python 3.12+ 移植版本，通过扫描 `memory/` 目录中的运行时日志、错误模式、用户反馈，从本地 GEP 资产库中选择匹配的 Gene 或 Capsule，生成一个协议约束的 GEP prompt，并记录可审计的 EvolutionEvent。

主要能力（Python 版本）：

- **CLI 入口** (`src/evolver/cli.py`): 支持 `evolver`、`evolver --loop`、`evolver --review`、`evolver solidify` 等命令。
- **进化流水线** (`src/evolver/evolve/runner.py` + `src/evolver/evolve/pipeline/`): 6 阶段流水线 (Preflight → Collect → Signals → Hub → Enrich → Select → Dispatch)。
- **GEP 核心** (`src/evolver/gep/`): 资产存储 (genes/capsules/events JSON/JSONL)、solidify、A2A 协议、本地状态、git 回滚、技能蒸馏/发布。
- **本地代理** (`src/evolver/proxy/`): 监听 `127.0.0.1:19820` 的 HTTP 代理，通过本地 JSONL mailbox 缓冲所有 Hub 流量，处理心跳、认证、任务同步、SSE、模型路由。
- **ATP 市场** (`src/evolver/atp/`): Agent Transaction Protocol，支持消费/商家代理、下单/交付/验证。
- **IDE 适配器** (`src/evolver/adapters/`): 为 Cursor、Claude Code、Codex、Kiro、OpenCode 安装 hooks。
- **WebUI** (`src/evolver/webui/`): 本地只读仪表盘，监听 `127.0.0.1:19821`。
- **生命周期/运维** (`src/evolver/ops/`): 健康检查、清理、自修复、创新触发器。

### 1.2 移植目标

创建 `./evolver.py/`，实现：

1. **CLI 行为等价**: 相同的命令、参数、环境变量、退出码。
2. **数据格式等价**: 完全兼容 `genes.json`/`genes.jsonl`、`capsules.json`/`capsules.jsonl`、`events.jsonl`、`memory_graph.jsonl`、`evolution_solidify_state.json` 等。
3. **文件路径等价**: 相同的默认路径解析（`~/.evomap/`、`~/.evolver/`、`.evolver/`、`memory/`、`assets/gep/`）。
4. **核心流水线等价**: signals → collect → hub → select → enrich → dispatch 的上下文传递和副作用。
5. **本地 Proxy 等价**: 路由、mailbox、sync engine、heartbeat、模型路由。
6. **ATP 等价**: 消费/商家代理、CLI 子命令、账本格式。
7. **适配器等价**: 生成的 hooks 文件与 Node 版本输出一致。
8. **测试等价**: 使用 `pytest` 重新实现关键测试，确保行为一致。

### 1.3 关键约束

- **Python 3.12+**: 使用 `asyncio.TaskGroup`、类型参数语法 (`list[int]`)、`tomllib` 等特性。
- **uv 包管理**: `pyproject.toml` 使用 `[project]` + `[dependency-groups]`，可执行 `uv run evolver`。
- **跨平台**: Windows / macOS / Linux，正确处理路径、进程锁、信号差异。
- **核心文件混淆**: 原始 Node.js 版本中的 `src/evolve.js`、`src/evolve/pipeline/*.js`、`src/gep/a2aProtocol.js`、`src/gep/solidify.js` 等被 `javascript-obfuscator` 混淆。Python 移植基于测试契约 + 原始 Node.js 版本调用点 + 公开接口，重构实现而非逐行翻译。

### 2.3 核心环境变量（完全保留）

#### 身份与 Hub

- `A2A_NODE_ID` — 节点身份
- `A2A_HUB_URL` — Hub URL，默认 `https://evomap.ai`
- `A2A_NODE_SECRET` — 共享 secret（文件优先）
- `GITHUB_TOKEN` — GitHub API token

#### 路径与工作区

- `EVOLVER_REPO_ROOT` — 覆盖仓库根
- `EVOLVER_HOME` — 每用户状态目录，默认 `~/.evomap`
- `OPENCLAW_WORKSPACE` — 工作区根覆盖
- `MEMORY_DIR` — memory 目录
- `EVOLUTION_DIR` — 进化状态目录
- `GEP_ASSETS_DIR` — GEP 资产目录
- `SKILLS_DIR` — 技能目录
- `EVOLVER_LOGS_DIR` — 日志目录
- `EVOLVER_SESSION_SCOPE` — 会话作用域子目录
- `AGENT_NAME` / `AGENT_SESSIONS_DIR` — OpenClaw 会话解析

#### 运行时行为

- `EVOLVE_STRATEGY` — 进化策略 (`balanced`/`innovate`/`harden`/`repair-only`/`early-stabilize`/`steady-state`/`auto`)
- `EVOLVE_BRIDGE` — 是否启用工作树突变
- `EVOLVER_ROLLBACK_MODE` — 回滚策略 (`stash`/`hard`/`none`)
- `EVOLVER_LLM_REVIEW` — LLM 预审
- `EVOLVER_VALIDATOR_ENABLED` — 验证者角色
- `WORKER_ENABLED` — 广播 worker 能力
- `EVOMAP_PROXY` / `EVOMAP_PROXY_PORT` — 代理开关/端口
- `EVOLVER_WEBUI_PORT` — WebUI 端口

#### 时间/限制（关键）

- `HEARTBEAT_INTERVAL_MS` = 360000
- `EVOLVER_HEARTBEAT_TIMEOUT_MS` = 10000
- `EVOLVER_MIN_SLEEP_MS` = 2000
- `EVOLVER_MAX_SLEEP_MS` = 300000
- `EVOLVER_MAX_CYCLES_PER_PROCESS` = 100
- `EVOLVER_MAX_RSS_MB` = 500
- `EVOLVER_CYCLE_TIMEOUT_MS` = 2700000 (45 min)
- `EVOLVER_VALIDATION_TIMEOUT_MS` = 180000
- `EVOLVER_CAPSULE_MAX_CHARS` = 8000
- `EVOLVER_PROMPT_MAX_CHARS` = 24000

完整列表见原始 Node.js 版本的 `src/config.js`，Python 版本（`src/evolver/config.py`）完整复刻。

### 2.6 进化流水线阶段

```
Preflight (guards.runPreflightChecks)
    ↓ abort? return
Collect (readMemorySnippet, readUserSnippet, readRealSessionLog, getMutationDirective, checkSystemHealth)
    ↓
Signals (extractSignalsStage, shouldSkipHubCalls)
    ↓ genes, capsules, signals, skipHubCalls
Hub (hubCoordinate)
    ↓ activeTask, hubLessons, lastHubFetchMs
Enrich (enrich)
    ↓ observations, memoryAdvice, hubHit, recentFailedCapsules
Select (selectAndMutate, computeAdaptiveStrategyPolicy)
    ↓ selectedGene, mutation, strategyPolicy, hypothesisId
Dispatch (dispatch)
    ↓ console output / file writes / sessions_spawn(...) / SOLIDIFY REQUIRED
```

## 3. Python 版本技术选型

### 3.1 包管理

- **工具**: `uv` (>=0.5)
- **配置**: `pyproject.toml` 使用标准 `[project]` 元数据
- **依赖组**: `[dependency-groups]` 区分 `dev` / `test`
- **脚本**: `[project.scripts]` 注册 `evolver = "evolver.cli:main"`
- **入口**: `src/evolver/__main__.py` 支持 `python -m evolver`

### 3.2 核心依赖

| 用途 | 包 | 版本 |
|---|---|---|
| CLI 解析 | `click` 或 `argparse` | latest |
| 异步运行时 | `asyncio` (stdlib) | 3.12+ |
| HTTP 客户端/服务器 | `httpx` + `fastapi` / `aiohttp` | latest |
| 数据验证 | `pydantic` >=2 | latest |
| 设置管理 | `pydantic-settings` | latest |
| 环境变量 | `python-dotenv` | latest |
| 加密 | `cryptography` | latest |
| Git 操作 | `GitPython` (可选) + `subprocess` | latest |
| 文件锁 | `filelock` | latest |
| 进程信息 | `psutil` | latest |
| 测试 | `pytest`, `pytest-asyncio`, `pytest-cov` | latest |
| 代码质量 | `ruff`, `mypy` | latest |

### 3.3 为什么不选某些包

- **不选 `requests`**: `httpx` 同时支持 sync/async、连接池、SSE、HTTP/2，等价替换 Node `undici`。
- **不选 `flask`**: FastAPI 原生支持 async，与 Proxy 的心跳/sync engine 更契合。
- **不选 `trio`/`anyio`**: 保持简单，Python 3.12 `asyncio` 已足够。
- **不选 `sqlalchemy` / `sqlite`**: 原始项目使用 JSON/JSONL，保持文件语义等价。

## 4. 模块级移植策略

### 4.1 CLI (`cli.py`)

- 使用 `argparse`（保持与 Node 版本一致的命令名和参数）。
- 顶层函数 `main()` 解析 `sys.argv[1:]`。
- 子命令映射到 `evolver.commands` 模块中的函数。
- `--loop` 模式使用 `asyncio.run()` 驱动 daemon loop。
- 处理 `.env` 加载顺序：cwd → repo root（完全复刻 #460 / #526 逻辑）。

### 4.2 配置 (`config.py`)

- 使用 `pydantic-settings` 定义 `EvolverConfig`。
- 每个字段保留原始环境变量名和默认值。
- 提供 `env_int`, `env_positive_int`, `env_float`, `env_str` 工具函数，行为与 Node 版本一致。
- `resolve_hub_url()` 函数复刻 URL 解析和 TLS 强制逻辑。

### 4.3 路径 (`gep/paths.py`)

- 使用 `pathlib.Path`。
- 函数签名保持与 Node 版本一致：`get_repo_root()`, `get_workspace_root()`, `get_evolution_dir()`, `get_gep_assets_dir()`, 等。
- `.git` 向上遍历时注意 Windows 根目录边界。
- `workspace-id` 文件生成 32 字符 hex。

### 4.4 Git 操作 (`gep/git_ops.py`)

- 所有命令使用 `subprocess.run(['git', ...])`，禁用 shell。
- `run_cmd()` / `try_run_cmd()` 包装器。
- `rollback_tracked()` 根据 `EVOLVER_ROLLBACK_MODE` 使用 `git stash push --include-untracked` 或 `git reset --hard`。
- `CRITICAL_PROTECTED_PREFIXES` / `CRITICAL_PROTECTED_FILES` 完全复刻。

### 4.5 数据模式 (`gep/schemas/`)

- 使用 Pydantic v2 `BaseModel`。
- `Gene`, `Capsule`, `Task` 模型字段、默认值、验证器完全复刻。
- `create_gene(partial)` / `validate_gene(g)` 函数签名等价。
- 保留 `VALID_CATEGORIES`, `VALID_ROUTING_TIERS` 等枚举常量。

### 4.6 资产存储 (`gep/asset_store.py`)

- 使用 `filelock.FileLock` 实现 `with_file_lock()`。
- JSON 文件读写使用临时文件 + `os.replace()`（原子重命名）。
- JSONL 使用追加模式 `a` 打开。
- 大文件 tail read 使用 seek 实现。
- 完全兼容 `genes.json`/`genes.jsonl`、`capsules.json`/`capsules.jsonl`、`events.jsonl` 格式。

### 4.7 信号 (`gep/signals.py` + `evolve/pipeline/signals.py`)

- `gep/signals.py`:
  - 三层提取：regex → keyword scoring → LLM fallback。
  - 保留所有信号名称和触发条件（`log_error`, `recurring_error`, `memory_missing`, `user_feature_request`, `perf_bottleneck`, `capability_gap`, `repair_loop_detected`, `evolution_saturation`, `force_steady_state`, 等）。
- `evolve/pipeline/signals.py`:
  - `extract_signals_stage(ctx)` 返回更新后的 context。
  - `should_skip_hub_calls(signals)` 复刻饱和门控逻辑。

### 4.8 进化流水线核心（混淆模块重构）

#### `evolve.py::run()`

```python
async def run() -> None:
    ctx = build_initial_context()
    preflight = await guards.run_preflight_checks()
    if preflight.abort:
        return
    ctx = pipeline.collect_phase(ctx)
    ctx = await pipeline.signals_phase(ctx)
    ctx = await pipeline.hub_phase(ctx)
    ctx = await pipeline.enrich_phase(ctx)
    ctx = await pipeline.select_phase(ctx)
    ctx = await pipeline.dispatch_phase(ctx)
```

#### `evolve/pipeline/collect.py`

- `read_memory_snippet()`: 读取 `MEMORY.md`，缺失返回 `[MEMORY.md MISSING]`。
- `read_user_snippet()`: 读取 `USER.md`。
- `read_real_session_log()`: 扫描平台会话目录。
- `get_mutationDirective(log)`: 解析 `recommended_intent` 和 `stability`。
- `check_system_health()`: 返回 Python 版本和运行时间。
- `format_cursor_transcript(raw)`: 清洗转录。

#### `evolve/pipeline/hub.py`

- `hub_coordinate(ctx)`: 如果 `skip_hub_calls` 则跳过。
- 调用 `task_receiver.fetch_tasks()`。
- 调用 `a2a_protocol.consume_hub_events()`。
- 返回 `active_task`, `hub_lessons`, `last_hub_fetch_ms`。

#### `evolve/pipeline/enrich.py`

- 调用 `memory_graph_adapter.record_signal_snapshot()` / `record_outcome()`。
- 调用 `hub_search` 搜索匹配资产。
- plateau 检测：≥10 连续失败 → `IS_RANDOM_DRIFT=True`, `plateau_pivot_required`。
- 5–9 连续失败 → `plateau_pivot_suggested`。

#### `evolve/pipeline/select.py`

- 信号匹配 → 选择 Gene / Capsule。
- 写入 hypothesis/attempt 事件到 memory graph。
- 返回 `selected_gene`, `strategy_policy`, `mutation`, `hypothesis_id`。

#### `evolve/pipeline/dispatch.py`

- 调用 `solidify.write_state_for_solidify()` 写入 `evolution_solidify_state.json`。
- 如果 `bridge_enabled`，调用 `bridge.write_prompt_artifact()` 并输出 `sessions_spawn(...)`。
- 否则输出 `BUILT_PROMPT` / `REUSE_PROMPT` + `SOLIDIFY REQUIRED`。
- 记录 `asset_call_log`。

### 4.9 GEP 核心混淆模块重构策略

由于原始 Node.js 版本的多个核心模块被混淆，无法逐行翻译。采用**契约驱动重构**：

1. 阅读对应测试文件，提取输入/输出契约。
2. 阅读原始 Node.js 版本中的调用点（如 `index.js`），或 Python 版本中的对应调用点（如 `cli.py`），明确 public API。
3. 使用最简实现满足契约，后续迭代细化。
4. 保留相同的数据结构和副作用。

## 6. 关键等价性保证

### 6.1 数据文件互操作

Python 实现必须能读取 Node 版本生成的文件，Node 版本也必须能读取 Python 版本生成的文件。保证措施：

- JSON/JSONL 使用相同字段名和默认值。
- 时间戳统一使用 ISO 8601。
- ID 生成使用 UUIDv7（与 Node `crypto.randomUUID` 一致）。
- SHA-256 canonicalization 完全相同。

### 6.2 环境变量

所有环境变量名、默认值、优先级完全复刻。`config.py` 提供单一可信源。

### 6.3 CLI 行为

命令名、参数、退出码、stdout/stderr 格式完全复刻。测试中使用 golden output 对比关键命令。

### 6.4 文件路径

默认路径解析逻辑完全复刻。通过设置 `EVOLVER_REPO_ROOT`, `MEMORY_DIR`, `EVOLUTION_DIR` 等环境变量，测试可以重定向到临时目录。

### 6.5 异步模型

Node 的 `async/await` + `Promise.race` 对应 Python 的 `async/await` + `asyncio.wait_for` + `asyncio.TaskGroup`。硬超时、取消、并发控制需仔细对齐。

### 6.6 进程管理

- Windows: 跳过 detached spawn，依赖外部 supervisor。
- macOS/Linux: 支持自杀重生 (`spawnReplacementProcess`)，使用 `subprocess.Popen(detach=True)`。
- 单实例锁：使用 PID + mtime 租约机制。

### 6.7 锁与并发

- 资产存储使用 `filelock`（跨进程）。
- Python 内部使用 `asyncio.Lock`（协程级）。
- JSON 写入使用 `tmp + os.replace`（原子）。

## 8. 风险与注意事项

### 8.1 混淆核心

原始核心算法文件被混淆，无法直接翻译。风险在于重构实现可能与原始行为存在细微差异。缓解：

- 严格遵循测试契约。
- 记录每个重构模块的假设和不确定点。
- 使用模糊测试/属性测试覆盖边界情况。

### 8.2 平台差异

- Windows 不支持 `SIGCONT`、`fork()`、`fcntl`。
- Windows `os.replace()` 与 Linux 行为略有不同（不能替换已打开文件）。
- 路径大小写不敏感问题。

缓解：

- 抽象平台差异到 `evolver.platform` 模块。
- CI 覆盖 Windows + Linux + macOS。

### 8.3 uv/Python 3.12 兼容性

- 确保所有依赖支持 Python 3.12+。
- 避免使用 `asyncore`、`imp` 等已移除模块。

### 8.4 性能

- JSONL 大文件读取使用 seek/tail，避免全量加载。
- Proxy 的 mailbox 使用批量 poll，避免高频率磁盘 IO。
- 心跳和 sync 定时器使用 `asyncio`，避免线程开销。

### 8.5 安全

- 所有 shell 命令禁用 `shell=True`。
- 验证命令白名单限制可执行文件。
- sanitize 模块扫描环境变量值反向外泄。
- TLS 强制默认开启，仅 `EVOMAP_HUB_ALLOW_INSECURE=1` 可绕过。
