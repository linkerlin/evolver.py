## 1. 对比方法论

1. **Git 历史比对**：提取 evolver/ 近 4 周 39 次发版的 `--stat`，定位高变更密度文件。
2. **源码树映射**：将 Node.js `src/**/*.js`（camelCase）映射到 Python `src/evolver/**/*.py`（snake_case），逐文件比对**存在性**与**行数比**（非混淆文件可比；混淆文件借助测试文件推断行为契约）。
3. **测试契约对标**：evolver/ 核心模块经 `javascript-obfuscator` 重度混淆（单行压缩），但其对应 `test/*.test.js` 为明文，以测试行数/断言数作为行为覆盖度代理。
4. **文档交叉验证**：evolver.py 的 `TODO.md`（1605 行，2026-06-11 更新）与 `README.md`/`AGENTS.md` 提供了模块自报状态，本方案以**实际文件行数**校正其自报值。

> ⚠️ 注意：evolver/ 的 `src/gep/*.js`（除少量文件）均为混淆单行，无法逐行比对实现深度。下文凡标注"Node 行数"来自**非混淆**文件（atp/、proxy/、adapters/、ops/、forceUpdate、部分 gep 测试），这些是可信的。

### 2. 各差距行为契约（取自明文测试）

#### G10.1 — 多源轨迹导出（最大缺口）

**输入源**（每种皆有专项测试）：
- Proxy trace JSONL：**AES-256-GCM 加密**，密钥由 `sha256('evomap-proxy-trace-v1:'+nodeSecret)` 派生；或 **RSA-wrapped hub-key envelope**（`--hub-private-key`）；行带 `secret_version` 选 keyring 条目，失败回落 nodeSecret；`--allow-partial` 导出明文行
- **Codex rollout** session JSONL → runtime session trajectory
- **Claude Code transcript** JSONL
- **OpenAI Responses**：`previousResponseId` 链合并、ambiguous owner 不合并、跨显式 session 不合并
- **OpenAI Chat Completions**：tool-call argument delta 流式重建
- **Anthropic native stream**：`tool_input_json_delta` 流式重建
- 通用 OpenAI messages / card(prompt/candidates/tools/meta + reasoning_content)
- **Cursor vscdb**（SQLite）、**Gemini CLI**、**Gemini Gateway**、**Kimi Wire**

**安全契约**：导出前**脱敏** runtime session bodies/tool inputs/errors/summaries；保留 system prompt 与 envelope 元数据；**原子写 mode 0600、不跟随预置 symlink**（PR #）；**fails-closed**：无法解密/无法识别格式即失败，不静默丢行。

**evolver.py 落点**：新建 `gep/trajectory/` 子包（`exporter.py`、`sources/`、`crypto.py`、`redact.py`），`evolver trajectory export [--input] [--output] [--hub-private-key] [--node-secret] [--node-secret-keyring] [--allow-partial]` CLI。

#### G10.2 — Solo 模式

`--solo` 路由进 loop 但强制：
1. **断网（breaker）**：即使 `A2A_HUB_URL=http://...` 也**覆盖为空**（"no escape valve"）；不启动 heartbeat/SSE/Proxy。
2. **禁 Validator**：即使 `EVOLVER_VALIDATOR_ENABLED=true` 也不启 ValidatorDaemon。
3. **禁 ATP**：即使 `EVOLVER_ATP=on`/`EVOLVER_ATP_AUTOBUY=on` 也覆盖关闭，startup 与 in-cycle（guards.py）双路拦截。
4. banner：`[Solo] Mad Dog · 受约束的野性模式已启动`、`断网 · 禁ATP`、`仅本机 git 可追踪`。
5. 新 env：`EVOLVER_CAFFEINATE`、`EVOLVER_SUICIDE`、`EVOLVER_CYCLE_TIMEOUT_ENABLED`、`EVOLVER_MIN/MAX_SLEEP_MS`、`EVOLVE_PENDING_SLEEP_MS`、`EVOLVER_MAX_CYCLES_PER_PROCESS`、`EVOLVER_LOCK_DIR`、`EVOLVER_SETTINGS_DIR`。
6. **gitGuard**：仅本机 git 可追踪，无网络推送。

**evolver.py 落点**：新建 `solo/` 子包（`breaker.py`、`git_guard.py`），CLI 加 `--solo` 分支（在 `cli.py` 早于 env 加载后、命令分发前注入覆盖），runner/guards 接入 solo 守卫。

#### G10.3 — CLI Contracts（`reuse`/`publish` v1）

**`reuse.v1`**（`evolver reuse --id <asset_id> [--json]`）：
- 接受 `--id`，拒绝缺 id / 位置 id / 未支持 flag / 缺 `--json`（皆 fail-closed）
- 存 Hub 资产时**剥离传输元数据**；写本地前**附 provenance**，provenance 失败即 fail-closed
- 本地 store 失败时**回滚已附 provenance**
- 拒绝"同 local id 不同 asset_id"；允许幂等"同 local id 同 asset_id"
- 拒绝 `asset_id ≠ 内容哈希` 的 Hub 资产；missing → `not_found`
- node-scoped fetch 用 **node secret**（OAuth 在场亦然）

**`publish.v1`**（`evolver publish <refs> [--json] [--dry-run]`）：
- 拒绝 bare Gene/Capsule、拒绝多 bundle 资产、拒绝位置参数、拒绝缺 `--json`
- dry-run：调 validate + 给 estimated credits，**不创建持久 node id**；quality 失败 → `blocked`
- fail-closes 空 validate 响应；解析稳定 Hub reason（非 not_found/quality fallback）
- **密钥脱敏 gate**：redact 字面 node secret（stdout JSON + dependency stderr）、嵌套 secret key/value；hard-reject 残留泄漏（忽略 legacy warn/off）
- **重哈希 + 重签** payload 后实际发布；`already_published` 200 → 幂等成功
- node-scoped validate/publish 用 node secret，拒绝 OAuth-only

**evolver.py 落点**：新建 `gep/cli_contracts.py`（统一 `parse_reuse_args`/`parse_publish_args`/`build_publish_bundle`/`run_reuse`/`run_publish`），把现散落逻辑收敛并补 provenance/redact-gate/rehash-resign；CLI `reuse`/`publish` 子命令改调它。

#### G10.4 — GEP Recipe 组合（Skill→Recipe）

`normalizeManifest(paths, {title})` + `composeRecipeFromSkills(manifest, {title,description,pricePerExecution,repoRoot,publish,strict})`：
- manifest：`{title, description, price_per_execution, max_concurrent, steps:[{skill_path, optional, condition, position}]}`
- 每步：水解 Skill（解析 `SKILL.md` frontmatter + `## Validation`）→ Gene；在 repoRoot 跑 validation 命令（**allow-list: node/npm/npx**）；strict 模式无 fallback（缺脚本→真实 exit 1）
- 组合 Recipe、构建 asset_ids、`publish:true` POST 到 market，`--no-publish` dry-run
- `A2A_NODE_ID` = recipe owner

**evolver.py 落点**：新建 `gep/skill2recipes.py` + `gep/skill2gep_audit.py` + `scripts/skill2recipes.py` + `examples/recipe.manifest.json`；**保留现有 `recipe/`（项目模板）**，二者命名区分（建议 GEP Recipe 走 `gep/` 下，模板 recipe 保留 `recipe/`，CLI 分别 `evolver recipe`（模板）与 `evolver skill2recipe`（GEP 流水线））。

#### G10.5 — Host Error Classifier (#571)

`isHostClientError(s)` + 非全局 `HOST_PROVIDER_ERR_RE`：
- **匹配**：`invalid_api_key`、`insufficient_quota`、`HTTP 400/401/403`、`rate limit exceeded`、`MaxTokens invalid`
- **不匹配**：`AssertionError`、`TypeError`、refactor 行数、空/null
- 集成进 `extract_signals`：host 4xx streak → 出 `host_llm_client_error` 信号，**抑制** `ban_gene`/`failure_loop_detected`（不归因 Gene、不累计失败 streak）

**evolver.py 落点**：新建 `gep/host_error_classifier.py`（≈35 行），`gep/signals.py` 接入分类。
